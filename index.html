Instrucciones

Para llevar a cabo este diseño web colaborativo debéis tener en cuenta las siguientes instrucciones

Git

Cada apartado tiene su/s página/s web propia/s. Además, deberéis trabajar cada uno de estos apartados en una rama individual de git, para evitar, en la medida de lo posible, que se produzcan conflictos a la hora de mezclar el código.

Estilos

Aunque una de las características de Bootstrap es que los estilos se establecen desde el código html a través de las clases, es posible que necesitéis añadir estilos, para ello utilizaréis un archivo propio para cada página web.

Metodología cooperativa

La evaluación de esta tarea va a ser global, esto quiere decir que vuestro éxito es el éxito de los demás, y vuestro fracaso el fracaso de los demás. Tanto los aspectos de diseño como de maquetación deben ser consensuados por todos, y aplicados a los diferentes contenidos que correspondan a cada uno. Además, debéis tener en cuenta que para la aplicación de los distintos contenidos, deberéis ayudar a vuestros compañeros a entender y aplicar los aspectos que os hayan correspondido a cada uno de vosotros desarrollar.

Reparto de tareas

Os corresponde a vosotros repartir las tareas. La única restricción es que cada uno de vosotros será responsable de dos apartados, tanto de desarrollarlos como de ayudar al resto de compañeros a que los entiendan e integren en sus respectivas secciones.

Diseño

El diseño proporcionado es solo una idea, sois libres de cambiarlo por completo, lo único que debe permanecer es la estructura (el index debe contener las instrucciones y un enlace al índice de contenidos).

Ejemplos

Utilizad distintos componentes de maquetación para distinguir las explicaciones de los ejemplos. La distribución y formato es libre, aunque debéis procurar que la página web guarde una coherencia de estilos.
